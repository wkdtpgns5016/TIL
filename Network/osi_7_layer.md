# 🌐 네트워크 - OSI 7계층

## 📌 1. OSI 7계층 개요
* **정의**: 국제표준화기구(ISO)에서 제정한 네트워크 통신 표준 모델
* **핵심 흐름 (캡슐화 및 디캡슐화)**:
  * **송신 측**: 데이터가 **상위 계층(7층) → 하위 계층(1층)**으로 내려가며 헤더가 붙는 **캡슐화(Encapsulation)** 진행
  * **수신 측**: 데이터가 **하위 계층(1층) → 상위 계층(7층)**으로 올라가며 헤더를 벗는 **디캡슐화(Decapsulation)** 진행

---

## 🔍 2. 계층별 상세 역할 및 특징



### 7계층: 응용 계층 (Application Layer)
* **주요 역할**: 사용자와 가장 밀접한 계층으로, 네트워크 응용 프로그램(웹 브라우저, 이메일 등)이 네트워크 서비스에 접근할 수 있도록 인터페이스를 제공합니다.
* **데이터 단위**: 데이터 (Data)
* **핵심 프로토콜**: `HTTP`, `FTP`, `SMTP`, `DNS`

### 6계층: 표현 계층 (Presentation Layer)
* **주요 역할**: 데이터의 형식(Format)을 정의하고 번역, 암호화, 압축을 수행하여 서로 다른 시스템 간에 데이터의 일관성을 유지합니다.
* **데이터 단위**: 데이터 (Data)
* **핵심 요소**: `JPEG`, `MPEG`, `SSL/TLS`

### 5계층: 세션 계층 (Session Layer)
* **주요 역할**: 통신 시스템 간의 연결(세션)을 설정, 유지, 동기화하고 종료하며 데이터 교환을 관리합니다.
* **데이터 단위**: 데이터 (Data)
* **핵심 요소**: `API`, `Socket`

---

### 4계층: 전송 계층 (Transport Layer)
* **주요 역할**: 종단(End-to-End) 간 사용자들에게 신뢰성 있는 데이터 전송을 담당하며, 흐름 제어 및 오류 제어를 수행합니다.
* **데이터 단위**: 세그먼트 (Segment)
* **핵심 프로토콜**: `TCP`(신뢰성 보장), `UDP`(빠른 전송) / 포트 번호

### 3계층: 네트워크 계층 (Network Layer)
* **주요 역할**: 데이터를 목적지까지 가장 빠르고 안전하게 전달할 수 있는 경로를 지정(Routing)하고 전송합니다.
* **데이터 단위**: 패킷 (Packet)
* **핵심 장비/식별자**: 라우터(Router) / `IP 주소`

### 2계층: 데이터 링크 계층 (Data Link Layer)
* **주요 역할**: 물리적 연결을 통해 인접한 노드(장비) 간의 데이터 오류를 제어하고 안전한 흐름을 보장합니다.
* **데이터 단위**: 프레임 (Frame)
* **핵심 장비/식별자**: 스위치, 브리지 / `MAC 주소`

### 1계층: 물리 계층 (Physical Layer)
* **주요 역할**: 전기적, 기계적 신호(0과 1)를 사용하여 케이블이나 무선 매체를 통해 데이터를 물리적으로 전송만 합니다. (알고리즘이나 오류 제어 기능 없음)
* **데이터 단위**: 비트 (Bit)
* **핵심 장비**: 허브, 리피터, 케이블
