# 🌐 네트워크 - HTTP/HTTPS 및 HTTP 메서드

## 📌 1. HTTP와 HTTPS 프로토콜

### HTTP (HyperText Transfer Protocol)
* **정의**: 웹 브라우저(클라이언트)와 웹 서버 간에 텍스트, 이미지 등 데이터를 주고받기 위한 기본 통신 규약
* **포트 번호**: `80` 번 포트 사용
* **특징**: 데이터를 암호화하지 않은 평문 형태로 전송하므로, 중간에 패킷을 가로챌 경우 데이터가 그대로 노출될 위험이 있음

### HTTPS (HTTP Secure)
* **정의**: HTTP의 보안 취약점을 해결하기 위해, HTTP에 **SSL/TLS 암호화 프로토콜**을 추가한 보안 버전
* **포트 번호**: `443` 번 포트 사용
* **특징**: 클라이언트와 서버가 주고받는 모든 데이터를 암호화하여 전송하므로 스니핑(비밀번호, 개인정보 탈취)을 방지함

## 🔍 2. HTTP 메서드 (HTTP Methods)

### HTTP 메서드란?
* 클라이언트가 웹 서버에 요청(Request)을 보낼 때, 해당 리소스에 대해 **수행하고자 하는 목적이나 행동**을 서버에 알려주는 통신 방식



### 💡 주요 메서드 종류 및 특징

* **🟢 GET (조회)**
  * **목적**: 서버의 특정 데이터를 **조회(Read)**할 때 사용
  * **특징**: 서버의 데이터를 변경하지 않는 안전한 메서드이며, 데이터(조건 등)를 URL 끝의 파라미터(Query String)에 담아 전송함

* **🔵 POST (생성/처리)**
  * **목적**: 서버에 **새로운 리소스를 생성(Create)**하거나, 데이터를 전송하여 특정 처리를 요청할 때 사용
  * **특징**: 데이터가 URL에 노출되지 않고 HTTP 요청의 바디(Body)에 담겨 전송됨

* **🟠 PUT (전체 대체/수정)**
  * **목적**: 리소스를 **전체 대체 및 수정(Update)**할 때 사용
  * **특징**: 지정한 경로에 데이터가 이미 존재하면 해당 데이터를 통째로 완전히 덮어쓰고, 대상 리소스가 존재하지 않으면 새로 생성함

* **🟡 PATCH (부분 수정)**
  * **목적**: 리소스의 **일부만 부분 수정(Update)**할 때 사용
  * **특징**: PUT과 달리 리소스 전체를 바꾸는 것이 아니라, 변경이 필요한 특정 필드만 선택적으로 수정함

* **🔴 DELETE (삭제)**
  * **목적**: 서버의 특정 리소스를 **삭제(Delete)**할 때 사용
